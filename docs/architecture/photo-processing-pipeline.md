The AI photo-processing pipeline uses an asynchronous, event-driven architecture with Cloudflare R2 for uploads, RabbitMQ for messaging, and a .NET Worker Service to handle third-party Vision API calls, ensuring high scalability and non-blocking user experiences. The process features direct R2 uploads, database updates to PostgreSQL and Redis, and real-time WebSocket notifications to the Next.js client, with robust error handling via Dead Letter Exchanges.
