# Database Schema: AI Photo-Scoring (MVP Clean Version)

## Status
PROPOSED

## Context
To keep data footprints minimal, API speeds fast, and database storage costs low, we are reducing the schema to the bare minimum required to show a user their progress history and ensure basic legal compliance in Brazil.

---

## 1. Entity Relationship Model (Minimalist)
[users] ──(1:N)──> [photos] ──(1:1)──> [analysis_results]

## 2. Table Definitions (PostgreSQL)

### Table: `users`
Tracks the user's explicit legal consent for handling sensitive aesthetic/health photo data, as strictly required by **LGPD**.

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    lgpd_consent BOOLEAN NOT NULL DEFAULT FALSE,
    lgpd_consent_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### Table: `photos`
Manages references to files stored in Cloudflare R2. This acts as the source file tracker for the asynchronous processing pipeline.

```sql
CREATE TYPE processing_status AS ENUM ('PENDING', 'PROCESSING', 'COMPLETED', 'FAILED');

CREATE TABLE photos (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    r2_object_key VARCHAR(512) NOT NULL, -- The clean path file identifier in R2
    status processing_status NOT NULL DEFAULT 'PENDING',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### Table: `analysis_results`
Stores the exact output generated by the .NET worker from the Third-Party Vision API. Complex sub-metrics are flattened into a single visual score to minimize table complexity.

```sql
CREATE TABLE analysis_results (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    photo_id UUID NOT NULL UNIQUE REFERENCES photos(id) ON DELETE CASCADE,
    cellulite_grade INT NOT NULL, -- Integer from 1 to 4
    smoothness_score INT NOT NULL, -- Flattened visual score (0 to 100) for the UI chart
    clinical_justification TEXT NOT NULL, -- Written Portuguese text for the user dashboard
    analyzed_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

## 3. Why This Is Enough For The MVP

1. **No Redundant Bloat**: The NestJS BFF or .NET worker does not need to parse deeply nested JSON configurations inside PostgreSQL. Everything needed to render the Next.js UI chart is simple flat integers (`cellulite_grade`, `smoothness_score`) and text (`clinical_justification`).
2. **Built for Performance**: By avoiding dynamic JSONB searches or complex relationship arrays, read queries hitting the dashboard execute in microseconds.
3. **Automatic Clean-up**: If a user invokes their LGPD right to be forgotten and deletes their account, `ON DELETE CASCADE` instantly drops their photos and analysis results from the database, leaving zero loose ends.
